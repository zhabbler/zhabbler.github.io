Жабблер API

Нет ничего приятнее, чем видеть, как талантливые дизайнеры и инженеры используют Жабблер для создания совершенно новых форм творческого самовыражения. Мы приложили огромные усилия, чтобы сделать этот API достаточно функциональным и гибким для любых проектов, в которых вы будете его использовать.

Чтобы вызвать функцию, вам необходимо перейти на страницу {your_domain}/developer/api и ввести название функции, к примеру, функцию которая показывает все посты: {your_domain}/developer/api/Posts/get_all_posts?last_id=0

Практически все функции требуют специальный айди сессии (Как авторизовать пользователя и получить айди сессии?).

Sessions

create_session

Параметры: email, password

Создаёт айди сессии и возвращает его. 

{
    "session": "{session}"
}

remove_session

Параметры: session, token

Удаляет сессию (к примеру, это нужно для выхода из аккаунта).

get_user_by_session

Параметры: session

Возвращает данные аккаунта и профиля используя специальный айди сессии. 

{
    "id": 1,
    "nickname": "admin",
    "name": "Administrator",
    "biography": "",
    "profileImage": "{url}",
    "profileCover": "",
    "email": "admin@localhost.lh",
    "token": "{token}"
}

User

get_user_by_token

Параметры: token

Возвращает данные аккаунта и профиля используя токен. 

{
    "id": 1,
    "nickname": "admin",
    "name": "Administrator",
    "biography": "",
    "profileImage": "{url}",
    "profileCover": "",
    "email": "admin@localhost.lh",
    "token": "{token}"
}

get_user_by_nickname

Параметры: nickname

Возвращает данные профиля используя никнейм. 

{
    "id": 1,
    "nickname": "admin",
    "name": "Administrator",
    "biography": "",
    "profileImage": "{url}",
    "profileCover": ""
}

Posts

add

Параметры: session, post, urlid, contains (1 = пост имеет спойлеры), who_comment (1 = никто), who_repost (1 = никто), repost (айди поста для репоста), question (айди вопроса для ответа), tags

Добавляет пост.

get_all_posts

Параметры: last_id (последний айди поста, к примеру это нужно для бесконечной прокрутки)

Возвращает список всех постов. 

[
    {
        "id": 1,
        "userID": 1,
        "nickname": "admin",
        "profileImage": "{url}",
        "postContent": "<h1>Hello, world!</h1>",
        "liked": 0,
        "uploaded": "2024-08-11 00:00:00"
    }
]

get_post

Параметры: post_id

Возвращает данные поста используя айди поста. 

{
    "id": 1,
    "userID": 1,
    "nickname": "admin",
    "profileImage": "{url}",
    "postContent": "<h1>Hello, world!</h1>",
    "liked": 0,
    "uploaded": "2024-08-11 00:00:00"
}

get_comments

Параметры: id, session

Возвращает список всех комментариев под постом. 

[
    {
        "id": 1,
        "profileImage": "{url}",
        "nickname": "admin",
        "comment": "Hello!",
        "post_id": "{post_id}",
        "added": "2024-08-11 08:43:32"
    }
]

comment

Параметры: session, post_id, comment

Добавляет комментарий под постом.

check_post_settings

Параметры: for (должен быть равен comments или reposts), post_id

Используется для того, чтобы узнать какие настройки поста были применены для комментариев или репостов.
Возвращает false если действие запрещено.

check_post_existence

Параметры: post_id

Возвращает true если пост существует.
